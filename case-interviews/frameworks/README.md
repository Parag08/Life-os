# 🗂️ Case Interview Frameworks — Cheat Sheet

> Quick reference for the main frameworks. Don't memorize rigidly — adapt to the case.

---

## 🏗️ Universal First Step: The Issue Tree

Always start by defining: **"What drives [the problem]?"**

```
Problem
├── Driver A
│   ├── Sub-driver A1
│   └── Sub-driver A2
└── Driver B
    ├── Sub-driver B1
    └── Sub-driver B2
```

**Rule:** MECE at every level (Mutually Exclusive, Collectively Exhaustive)

---

## 💰 Profitability Framework

```
Profit = Revenue − Cost

Revenue = Price × Volume

Volume = Market Size × Market Share

Costs
├── Fixed (rent, salaries, depreciation)
└── Variable (COGS, shipping, commissions)
```

**Hypothesis ladder:**
1. Is it a revenue problem or cost problem?
2. If revenue: price or volume?
3. If volume: market demand or share?

---

## 🌍 Market Entry Framework

1. **Is the market attractive?** (size, growth, profitability)
2. **Can we compete?** (capabilities, differentiation)
3. **How should we enter?** (build, buy, partner)
4. **What are the risks?** (regulatory, competitive reaction)

---

## 📈 Growth Framework

```
Growth Options
├── Existing markets (penetration, price)
├── New segments (new geographies, new customers)
├── New products (innovation, adjacencies)
└── Acquisitions / partnerships
```

---

## 🤝 M&A Framework

1. **Strategic rationale** — why this deal?
2. **Target quality** — financials, market position, risks
3. **Synergies** — revenue upside + cost savings
4. **Integration risk** — culture, ops, systems
5. **Valuation** — is the price fair?

---

## ⚙️ Operations / Process Improvement

```
Capacity
├── Bottleneck identification (where is the constraint?)
├── Throughput analysis
└── Utilization rates

Cost / Efficiency
├── Lean / waste reduction
├── Automation opportunities
└── Outsourcing

Quality
└── Defect rates, returns, customer complaints
```

---

## 📐 Market Sizing (Fermi Estimation)

**Step-by-step:**
1. Define the thing being estimated clearly
2. Break it down: population → segment → penetration → frequency → spend
3. Do the math, narrate every step
4. Sense-check: does the number feel right?

**Example structure:**
```
Market size = Population × % addressable × penetration × avg spend
```

---

## 🗣️ Communication Principles

- **Lead with the answer** — BLUF (Bottom Line Up Front)
- **Use signposting** — "I'd like to look at three areas..."
- **Check in frequently** — "Does this structure make sense?"
- **Synthesize, don't summarize** — "Based on our analysis, the core issue is..."
